CSS Selector Demo

Below are three boxes showing grouping, chaining, and descendant selectors in action.

1) Grouping selectors (comma = “OR”)

Rule: .grouping-demo h1, .grouping-demo h2, .grouping-demo .subtitle { ... }

H1 styled via grouping

H2 styled via grouping

.subtitle styled via grouping

Expected: All three above share the same color and bottom border because they’re grouped with commas.

2) Chaining selectors (no space = “AND”)

Rules: .btn.primary { ... }, input[type="text"].error { ... }, .badge.pill { ... }


I match .badge.pill  |  I do NOT match (missing .pill)

Expected: Only elements that meet all chained conditions get styled.

3) Descendant selectors (space = “inside”)

Rules: .card em { ... }, nav a { ... }, and a direct-child contrast .card > h3.

  • Home
  • Products (current)
  • About

Direct child h3 (matches .card > h3)

Some text with an emphasized word (matches .card em).

Nested h3 (NOT a direct child)

Still inside the card, this em also matches because it’s a descendant.

Expected: All <em> inside .card get styled (any depth). The first h3 (direct child) is green; the nested one isn’t (because > targets direct children only). Links inside <nav> are styled; the current page link has a background.

Bonus: Direction matters for descendants

.card p means “p inside .card”. The reverse p .card would mean “.card inside p” (rare/invalid unless .card is on an inline like <span> inside the paragraph).
